Using Jira

What is Jira? Jira is a popular project management and issue-tracking tool used by software development teams to track bugs, tasks, and project progress. Our generator can pull requirements and user stories directly from Jira to create test cases.

How to get your Jira credentials:

  1. Jira URL: This is the main web address for your Jira instance (e.g., https://your-company.atlassian.net).
  2. Jira User: Use the email address associated with your Atlassian account.
  3. Jira API Token: This is a secure key used instead of your password. To get it:
    • Log in to your Atlassian account.
    • Click on your profile avatar and go to "Account Settings."
    • Navigate to the "Security" tab.
    • Under the "API token" section, click "Create and manage API tokens."
    • Click "Create API token," give it a name, and copy the token. **This token will only be shown once, so save it somewhere secure!**
  4. Item ID(s): Enter the specific Jira ticket IDs you want to analyze (e.g., PROJ-123). You can enter up to 10 items for batch processing.

Using Azure DevOps

What is Azure DevOps? Azure DevOps is a suite of tools for collaborative software development, including version control, pipelines, and work item tracking. Our generator connects to your Azure DevOps work items to analyze requirements and create tests.

How to get your Azure DevOps credentials:

  1. Azure DevOps URL: This is the URL for your organization (e.g., https://dev.azure.com/your-organization).
  2. Azure Organization: The name of the organization you are working in.
  3. Azure Project: The name of the specific project you are working on.
  4. Personal Access Token (PAT): A PAT is a secure, alternative password for authentication. To get it:
    • Log in to your Azure DevOps account.
    • Click the "User settings" icon in the top right corner and select "Personal access tokens."
    • Click "New Token."
    • Give your token a name, select the organization, and set an expiration date.
    • For **Scopes**, select the necessary permissions like "Work Items" and "Projects & Teams" to grant read access.
    • Click "Create" and copy the token. **Like the Jira token, you will only see this once!**
  5. Item ID(s): Enter the specific work item or user story IDs you want to analyze. You can enter up to 10 items for batch processing.

Best Practices

  • Use clear, well-structured titles and descriptions in Jira/Azure items to improve the quality of generated tests.
  • When using URLs, prefer publicly accessible pages with stable markup and minimal authentication gates.
  • For image-based generation, upload high-resolution screenshots that contain labels, controls, and visible states.
  • Leverage batch processing for similar items to keep formatting and tone consistent across test suites.

Security & Privacy

  • Jira tokens and Azure PATs should be created with least-privilege scopes and rotated periodically.
  • Credentials are used only for generating test cases and should not be embedded in shared links or exports.
  • Remove sensitive content from screenshots/URLs if they are not intended for broader sharing.

Troubleshooting

  • No Item IDs found: Verify credentials and organization/project names. Ensure items are visible to your token’s scope.
  • Charts not updating: Refresh the page. Legends are clickable; if a segment is hidden, click the label to reveal it.
  • Long generation time: Large batches and complex inputs may take longer. Progress indicators and retries are built in.
  • Export issues: Try generating again or exporting a smaller subset to isolate faulty records.
